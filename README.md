فاز 1: برسی و مستند سازی API های سمت سرور
در این مرحله، تمامی API های سمت بک‌اند پروژه از طریق محیط Swagger به‌صورت کامل بررسی و تست شدند. هدف این فاز، آشنایی دقیق با ساختار سرویس‌ها، نحوه ارسال درخواست‌ها و تحلیل پاسخ‌های دریافتی از سرور بود.

در گام اول، لیست کامل API ها استخراج و هر یک از سرویس‌ها از نظر ورودی‌ها (Parameters)، نوع درخواست (Method)، خروجی‌ها (Responses) و کاربرد عملکردی مورد بررسی قرار گرفتند. پس از انجام تست‌های لازم، اطلاعات مربوط به هر API به‌صورت ساختاریافته مستندسازی شد.

فاز ۲ — بررسی صفحات سمت کاربر و شناسایی API های فراخوانی‌شده

در این مرحله، تمامی صفحات سمت کاربر (Front-End) پروژه به‌صورت کامل بررسی شدند تا مشخص شود هر صفحه برای دریافت یا ارسال اطلاعات از چه API هایی استفاده می‌کند. هدف این فاز، درک دقیق ارتباط بین رابط کاربری و سرویس‌های بک‌اند و همچنین تحلیل نحوه تبادل داده‌ها در سناریوهای مختلف بود.

برای انجام این بررسی، از ابزار Inspect مرورگر (Developer Tools) استفاده شد. تمرکز اصلی بر روی تب‌های زیر بود:

Network: جهت مشاهده درخواست‌های ارسالی، نوع متدها (GET / POST / PUT / DELETE)، پارامترهای ورودی و پاسخ‌های دریافتی از سرور

Console: بررسی خطاهای احتمالی و لاگ‌های مرتبط با فراخوانی API ها

Application / Storage: تحلیل داده‌های ذخیره‌شده مانند Token ها و Session ها در LocalStorage یا Cookies

Headers & Payload: بررسی دقیق هدرها، احراز هویت (Authorization) و بدنه درخواست‌ها

در نتیجه این تحلیل، برای هر صفحه مشخص شد که چه API هایی در چه شرایطی فراخوانی می‌شوند و نحوه تعامل فرانت‌اند با بک‌اند به‌صورت کامل مستندسازی گردید. این مرحله نقش مهمی در تسلط بر ساختار ارتباطی پروژه و آماده‌سازی برای مراحل توسعه و اتصال API ها داشت.
